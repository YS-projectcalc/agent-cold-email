CLASS: test assertions gated on ambient developer-environment secrets. `@cloudflare/vitest-pool-workers` (pool/index.mjs → `wrangler.unstable_getMiniflareWorkerOptions` → `getBindings2` → `getVarsForDev`, wrangler cli.js:184529) reads `<wrangler.toml dir>/.dev.vars` and injects EVERY key as a `secret_text` binding on the test `env`. So a developer's locally-wired real key (ColdStart: a truthy `STRIPE_SECRET_KEY` in `apps/platform/.dev.vars`) silently flips a code path under test (`engine/billing.ts` `isRealSpendArmed`) → 5 `checkout.test.ts` simulated-mode tests fail. The suite is green ONLY in fresh worktrees because they copy the empty-key `.dev.vars.example` — result depends on ambient machine state.

**Why:** the pool exposes no lever to skip `.dev.vars` (its `getBindings2` `envFiles` arg is unset and not reachable via the plugin's public options). But the pool merges `options.miniflare.bindings` OVER the wrangler-derived `.dev.vars` bindings (miniflare `mergeWorkerOptions` → `Object.assign(wranglerBindings, userBindings)`), so `vitest.config.ts`'s `miniflare.bindings` always wins.

**How to apply / fix pattern (hermetic-by-default, NOT a per-var blocklist):** in `vitest.config.ts`, read the KEYS from `.dev.vars` (+ `.dev.vars.example`; values never logged — CLAUDE.md rule g) and CONSTRUCT `miniflare.bindings` = { every non-allowlisted ambient key → `null` } + { allowlist with fixed test values }. Neutralize to `null`, NOT `""`: `null` is in miniflare's binding `LiteralSchema` (JSON-valid) AND nullish, so it stays falsy for `Boolean`/`&&` gates AND falls through `?? default` reads (e.g. `tick.ts` `PUBLIC_BASE_URL`) exactly like an unset var; `""` breaks the `??` case; `undefined` fails miniflare's zod. A new env var (e.g. a REGISTRAR_* lane) is hermetic automatically because the sweep neutralizes ALL non-allowlisted ambient keys. Guard (failing-by-construction): a canary in `.dev.vars.example` (copied → `.dev.vars` in fresh worktrees) + a test asserting `env.<canary>` and every `// spend-arming` env.ts binding read falsy. ColdStart impl: `apps/platform/test/hermetic-env.ts` + `hermetic-env.test.ts` (commit 284b96b, branch worktree-test-env-hermetic-20260723).
