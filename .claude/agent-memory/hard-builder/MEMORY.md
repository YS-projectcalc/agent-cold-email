# hard-builder memory index

## Failure mechanisms (check before diagnosing a hard case)
- [coldstart-per-tick-recompute-clobbers-control-state](coldstart-per-tick-recompute-clobbers-control-state.md) — a per-tick refresh that recomputes a column (warmup daily_cap/status) silently wipes any control-loop override written to the SAME column; keep loop state in a SEPARATE column + have the refresh honor it (MIN).
- [coldstart-suspend-auth-split-do-vs-d1](coldstart-suspend-auth-split-do-vs-d1.md) — ColdStart: DO `tenant_profile.status='suspended'` does NOT lock the token (requireAuth reads D1 `tenants_index.status`); flip BOTH to truly disable a tenant, or a "suspended" tenant re-provisions.
- [backtick-inside-template-literal-sql](backtick-inside-template-literal-sql.md) — a backtick in a backtick-delimited SQL template literal (even in a `--` comment) ends the string → misleading TS1005 errors; never backtick-quote inside a template literal.
- [sandbox-port-masks-real-server-contract](sandbox-port-masks-real-server-contract.md) — ColdStart A5 spike: the in-process sandbox EmailPort hides 4 real-server obligations (reconstruct threadId from headers, set real RFC5322 Message-ID, at-least-once re-poll dedupe [events table has none], soft-vs-hard bounce) + SendEmailInput can't carry List-Unsubscribe; validate against a real SMTP/IMAP server before freezing a port.
- [async-tally-reset-on-triggering-action](async-tally-reset-on-triggering-action.md) — resetting a counter on the SAME action that async-feeds it (soft-bounce streak DELETE'd on every send, bounce polled AFTER) makes the threshold unreachable; reset only on an independent POSITIVE signal (reply), clear on permanent-suppress. A single-tick multi-lead fixture hid it — needs a real over-time single-lead trace.
- [dsn-arf-fields-live-in-mime-subparts](dsn-arf-fields-live-in-mime-subparts.md) — DSN/ARF report fields (Final-Recipient, Original-Rcpt-To, Status) live in MIME SUB-PARTS, not the outer header block; an RFC 5322 top-level header parser silently returns empty for them → use a whole-source line-anchored scan. (ColdStart engine classify.ts.)
- [persist-before-confirm-cross-boundary](persist-before-confirm-cross-boundary.md) — CLASS: a cursor/claim/status persisted BEFORE the cross-boundary effect (vendor call / HTTP response / DO commit) is confirmed → work silently lost or row stuck on crash. Fix: consumer-owned cursor advanced only inside the same txn as the confirmed work, OR persist-after-confirm. Members: engine poll cursor, tick send-on-'sending', idempotency 'pending', I3 billed-mailbox cred-push (record BEFORE push, F6).
- [json-store-corrupt-catchall-silent-empty](json-store-corrupt-catchall-silent-empty.md) — CLASS (F5): a JSON durable store whose loader catches ALL errors → empty silently drops state on corruption AND overwrites the only copy on next flush; split MISSING (ENOENT→empty) from CORRUPT (parse/shape error→THROW, refuse to boot). Members: engine-state.json, pushed-mailboxes.json.
- [period-keyed-counter-for-persistent-resource](period-keyed-counter-for-persistent-resource.md) — a per-period (monthly) ledger counter gating a PERSISTENT resource (plan-slot occupancy) resets each period while occupancy persists → over-allocation after rollover; keep occupancy counters account-wide/single-row + a durable per-item release marker (teardown reads sandbox, can't tell real from sandbox). (GA gate G4.)

## Test-env gotchas (ColdStart apps/platform vitest)
- [coldstart-vitest-binding-and-d1-isolation-gotchas](coldstart-vitest-binding-and-d1-isolation-gotchas.md) — a wrangler.toml binding IS bound in the vitest env (not undefined; asserting `.toBeUndefined()` throws RPC inspect errors) → test factory branches with fake envs; direct `env.DB` writes are NOT rolled back per-test → `beforeEach` DELETE for state-machine tables; `test/setup.ts` needs each new migration added by explicit `?raw` import.
- [vitest-pool-workers-ambient-devvars-leak](vitest-pool-workers-ambient-devvars-leak.md) — CLASS: the pool auto-loads `apps/*/.dev.vars` via wrangler and injects EVERY key as a test binding → a dev's ambient real secret (truthy STRIPE_SECRET_KEY) flips a behavior gate under test; fix = CONSTRUCT `miniflare.bindings` from an allowlist, neutralize non-allowlisted ambient keys to `null` (nullish+JSON-valid, not `""`/`undefined`); hermetic-by-default + canary guard.

## Build patterns / gotchas (ColdStart engine)
- [stripe-percent-coupon-prorates-at-unit-level](stripe-percent-coupon-prorates-at-unit-level.md) — a percent coupon discounts RECURRING lines via `discount_amounts` (gross line) but bakes the discount INTO the PRORATION line `amount` (desc "(with X% off)", discount_amounts=0), so `total != subtotal×(1−pct)`; assert the bumped recurring line's discount + a proration desc `/% off/i`. A decrease-no-credit test must run on a FRESH sub (an increase proration emits a "-Unused time" credit component). (quantity-billing Tier-2 gate.)
- [coldstart-authed-route-needs-path-pattern-and-zod-default-output-required](coldstart-authed-route-needs-path-pattern-and-zod-default-output-required.md) — a new authed route 500s (not 401) until its literal path is added to `AUTHED_PATH_PATTERNS` in index.ts (per-path requireAuth, never blanket `*`); AND `z.X().default(v)` makes the field REQUIRED in the inferred OUTPUT type → every direct-engine-call test must supply it.
- [coldstart-mailcomposer-single-builder-for-transports](coldstart-mailcomposer-single-builder-for-transports.md) — new send transports (Gmail/Graph API) must carry compliance headers byte-identically: reuse nodemailer's MailComposer as the ONE raw-message builder (deep import + @types both work); map all API failures to the SMTP path's UpstreamTransientError.
- [coldstart-env-example-gitignored-and-header-folding](coldstart-env-example-gitignored-and-header-folding.md) — `.env.example` is gitignored by `.env.*` (durable docs → README); long MIME headers fold across lines (assert fold-tolerantly, not single-line).
