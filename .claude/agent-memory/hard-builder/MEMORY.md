# hard-builder memory index

## Failure mechanisms (check before diagnosing a hard case)
- [coldstart-per-tick-recompute-clobbers-control-state](coldstart-per-tick-recompute-clobbers-control-state.md) — a per-tick refresh that recomputes a column (warmup daily_cap/status) silently wipes any control-loop override written to the SAME column; keep loop state in a SEPARATE column + have the refresh honor it (MIN).
- [coldstart-suspend-auth-split-do-vs-d1](coldstart-suspend-auth-split-do-vs-d1.md) — ColdStart: DO `tenant_profile.status='suspended'` does NOT lock the token (requireAuth reads D1 `tenants_index.status`); flip BOTH to truly disable a tenant, or a "suspended" tenant re-provisions.
- [backtick-inside-template-literal-sql](backtick-inside-template-literal-sql.md) — a backtick in a backtick-delimited SQL template literal (even in a `--` comment) ends the string → misleading TS1005 errors; never backtick-quote inside a template literal.
- [sandbox-port-masks-real-server-contract](sandbox-port-masks-real-server-contract.md) — ColdStart A5 spike: the in-process sandbox EmailPort hides 4 real-server obligations (reconstruct threadId from headers, set real RFC5322 Message-ID, at-least-once re-poll dedupe [events table has none], soft-vs-hard bounce) + SendEmailInput can't carry List-Unsubscribe; validate against a real SMTP/IMAP server before freezing a port.
- [async-tally-reset-on-triggering-action](async-tally-reset-on-triggering-action.md) — resetting a counter on the SAME action that async-feeds it (soft-bounce streak DELETE'd on every send, bounce polled AFTER) makes the threshold unreachable; reset only on an independent POSITIVE signal (reply), clear on permanent-suppress. A single-tick multi-lead fixture hid it — needs a real over-time single-lead trace.
- [dsn-arf-fields-live-in-mime-subparts](dsn-arf-fields-live-in-mime-subparts.md) — DSN/ARF report fields (Final-Recipient, Original-Rcpt-To, Status) live in MIME SUB-PARTS, not the outer header block; an RFC 5322 top-level header parser silently returns empty for them → use a whole-source line-anchored scan. (ColdStart engine classify.ts.)
- [persist-before-confirm-cross-boundary](persist-before-confirm-cross-boundary.md) — CLASS: a cursor/claim/status persisted BEFORE the cross-boundary effect (vendor call / HTTP response / DO commit) is confirmed → work silently lost or row stuck on crash. Fix: consumer-owned cursor advanced only inside the same txn as the confirmed work, OR persist-after-confirm. Members: engine poll cursor, tick send-on-'sending', idempotency 'pending'.
