ColdStart (`~/dev/coldstart/`) canonical living docs, per its `CLAUDE.md`: `README.md` (per level), `SPEC.md` (architecture), `ROADMAP.md` (build order/status — now the ledger under the ROADMAP Contract, see below), `ARCHITECTURE.md`, `HANDOFF.md` (resume-here state), `MEMORY.md` (repo build lessons), `ACTIVATION.md` (owner go-live checklist, once created). NEVER create new status/audit/critique/handoff-vN docs. One exception: adversarial panel verdicts are frozen records under `docs/adversarial/panel-NN/` (allowed — not living docs, don't route new content there). Research/decision provenance also gets frozen dated files under `docs/research/*-YYYY-MM-DD.md` — these are write-once, never edited after landing (even to fix a noted error — corrections get a pointer note elsewhere, e.g. ROADMAP, not an edit to the frozen file).

**ROADMAP Contract adopted 2026-07-13** (from the global `~/CLAUDE.md` § ROADMAP Contract, now live in this repo): ROADMAP.md keeps exactly TWO living sections, `## Now` (≤5 items, actively worked) and `## Open` (everything else, one line each: `- [ ] YYYY-MM-DD [ORDER|ASK|IDEA] <item>`, status tags like `[gated:founder]`/`[dark-unarmed]`). The OLD narrative "Session log" section is GONE from ROADMAP.md — it was migrated verbatim to `archive/ROADMAP-done.md` (append-only, dated; see `archive/INDEX.md` for pointers) as part of a 2026-07-13 `/handoff` restructure. New completed work no longer gets a narrative paragraph — it gets checked `- [x]` lines in `## Open` with a compact evidence pointer (commit hash, test count, verdict), and self-drains to the archive at the NEXT handoff. Do not re-add a narrative session-log section — that would be exactly the sprawl this Contract exists to kill. HANDOFF.md's "Where we are right now" is a snapshot (facts), "In flight / next" is action-oriented (what's running/blocked/next); "Resume" section is the founder-facing script — a bookkeeping brief that says "surgical edits to those sections only" should NOT touch Landmines/Key files even if related content would fit, unless explicitly told to.

Git in this repo: routinely a shared/live worktree across concurrent agent sessions (see ROADMAP-done.md archive, 07-09 note re: a concurrent `git add -A` sweeping up in-progress files) — historically bookkeeper briefs here were git-read-only, but a 2026-07-13 brief explicitly asked for a docs-only commit on main (message prefix "docs:") — commit when the brief says so, stay read-only otherwise. Watch for concurrent edits to ROADMAP.md/HANDOFF.md mid-session (hit once 2026-07-13 — Edit failed with "file modified since read," re-Read caught a sibling process's in-flight restructure).

**Split-ownership file technique (2026-07-14):** ACTIVATION.md is a canonical doc I'm allowed to touch, but it can arrive with an ALREADY-DIRTY working tree from a different lane (e.g. a builder had drafted a whole Gate-2 engine runbook, uncommitted, adjacent to the one line my brief authorized). `git checkout`/`stash` risks losing their in-flight text; committing the whole file risks landing content the brief explicitly said isn't ready. Fix: make my Edit directly on the live (dirty) file, then hand-build a minimal patch — `git diff -U0 -- FILE > full.patch`, mechanically extract (via `sed`, never retyped by hand — Unicode em-dashes/arrows in this repo's prose break hand-transcribed patches) just my target line plus ~3-4 lines of real surrounding context on each side (a 1-2 line minimal hunk reliably fails `git apply --check` with "patch does not apply" even when content matches byte-for-byte — use the full original hunk's context window), then `git apply --cached --check` then `--cached` (no working-tree flag) to stage ONLY that hunk into the index. `git diff --cached` afterward should show just my line; `git diff` (unstaged) should show just their untouched content. Commit, then `git status` confirms their hunk is still sitting dirty exactly as before. [[coldstart-doc-convention-map]]
