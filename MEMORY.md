- curl HEAD-vs-GET lesson (2026-07-24, SDN relay): a droplet `curl` against the Treasury SDN feed got a 200 on a HEAD probe but a 302 redirect on the actual GET — `curl` invocations that don't pass `-L`/`--location` (the push script originally didn't) will silently fetch a redirect-page body instead of the real file. Always test the ACTUAL request verb+redirect-following behavior you'll use in production, not a HEAD-only preflight — they can disagree.
- Trailing DOS-EOF-byte lesson (2026-07-24, SDN relay): the real Treasury SDN.CSV ends with a lone `0x1A` (legacy DOS EOF/SUB marker) after the last data row. A fail-loud CSV parser correctly rejected this as malformed on the very first live push. Fix: strip a trailing run of blank/whitespace lines and/or a single trailing `0x1A`, anchored strictly to the end of the file — an `0x1A` or blank line ANYWHERE else (i.e., touching real data) must still be treated as fatal corruption. Lesson: "tolerate the known-benign tail artifact, keep failing loud on everything else" is the right shape for real-world feed quirks — don't loosen the whole parser to work around one specific trailing byte.
